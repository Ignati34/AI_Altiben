1. Executive Summary

Altiben S.L. seeks to transition from traditional tax consultancy workflows to a digital-first ecosystem. This project involves the creation of a modern digital presence integrated with an intelligent client portal.

Core Vision

"Modernizing tax consultancy through the seamless integration of Artificial Intelligence and OCR technology, providing 24/7 client support and automated document handling while maintaining the highest security and accessibility standards."

2. User Personas

C

Clients (End Users)

Small business owners, freelancers, and private individuals. They require a frictionless way to submit documents, track tax obligations, and get instant answers to common fiscal queries.

A

Administrators (Staff)

Tax advisors and administrative staff. They need a dashboard to manage document queues, review OCR-extracted data, and monitor AI-human handoffs for complex queries.

3. Functional Requirements

FR-01 Multilingual Engine

The platform must support 17+ languages. Localization must include Catalan, Spanish, English, French, and German as priority tiers. Automatic browser detection and manual toggle required.

FR-02 AI Assistant & Voice

24/7 AI-powered chatbot with voice-to-text capabilities. It must be trained on Spanish/Catalan tax laws and provide real-time guidance on standard procedures.

FR-03 OCR Document Processing

Intelligent Document Processing (IDP) using OCR for receipts, invoices, and bank statements. System must extract VAT, total amount, provider ID, and date with >95% accuracy.

FR-04 Payment Integration

Subscription-based billing models. Integration with Stripe for credit card payments and SEPA mandates for recurring monthly fees.

4. Technical Requirements

Database Schema (Partial)

Table Name Primary Key Description
users user_id (UUID) Core user identity and authentication data.
documents doc_id (UUID) Stores metadata and OCR extraction results for uploads.
subscriptions sub_id (UUID) Active billing cycles and tier access levels.

API Architecture

  • Chatbot API: RESTful endpoints for message submission and response streaming.
  • Authentication: OAuth 2.0 and JWT (JSON Web Tokens) for secure session management.
  • Compliance: WCAG 2.1 Level AA Accessibility Compliance for all web interfaces.

6. Acceptance Criteria

  • System handles OCR for a 100-page PDF batch in under 60 seconds.
  • Chatbot accurately detects Catalan vs Spanish within the first two exchanges.
  • All payment transactions meet PCI-DSS compliance standards.
  • Audit logs are maintained for all document modifications.

7. Development Roadmap

Phase 1: Foundation (Weeks 1-4)

UI Implementation, Auth Flow, Database setup.

Phase 2: Core Features (Weeks 5-8)

Multilingual Engine, AI Assistant integration, Calendly link.

Phase 3: Advanced Functionality (Weeks 9-12)

Client Portal, OCR Processing for documents & receipts.

Phase 4: QA & Deployment (Weeks 13-16)

Security auditing, Performance optimization, Final launch.
